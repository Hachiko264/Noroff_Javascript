What is the DOM?
-Står for "Document Object Model"
- Litt vanskelig å forklare, men det er selve HTML-filen til
nettsiden som alt kobles til. Altså selve dokumentfilen.
- Man bruker document.body; eller title/url osv for å hente ut
informasjon om dokumentet som inneholder hele nettsiden.

List three methods to manipulate the DOM.
- endre elementer, endre klasser og endre attributer. 

What is an asynchronous call?
- at koden ikke venter på API signalet på å komme tilbake fra
serveren. Den blokkes heller ikke, og fortetter videre uten å vente. 
-Ulempen er at man ikke kan garantere at f.eks to alerts kommer i 
riktig rekkefølge. 

Briefly explain what the sources tab in Chromes 
Developers Tools does.
- Viser hvordan nettsider er bygd opp, endten full html (DOM), 
eller console for å se javascripten, eller sources hvor man ser 
mappestrukturen. Finnes også fler, men disse er de mest vanlige. 

What is a JavaScript promise?
- For bruk når du ikke vet verdien som skal brukes. f.eks pending,
fulfilled eller regjected. 

What is an API?
- Står for "Application Programming Interfaces"
- Klientdelen til Javascript
- Deles inn i browser og third party API. Hvor f.eks lydavspilling
i nettleseres er en browser-api, men twitter-plugins en 3rd party.
- Gjør nettsiden mye mer fancy.
- Gjør blandt annet 3Dgrafikk mye lettere.

What is a JavaScript Event?
- En hendelse på nettsiden. At man refresher siden. Klikker en knapp, osv.
- Kan aktivere seg selv, eller av andre. 

What is What is a JavaScript Event?
- At en event "bobler" utover i parent-objektene. 
F.eks p-div-form helt til html. evt definer når det stoppes.
